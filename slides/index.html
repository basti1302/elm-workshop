A Counter

(A Very Simple Elm App)

Counter App - main

main : Html a
main = text "Hello React Days!"


main : Program Never Model Msg
main =
    Html.beginnerProgram
        { model = model
        , view = view
        , update = update
        }

Counter App - Model


type alias Model = Int

model : Model
model = 0

Counter App - Messages


type Msg = Increment | Decrement

Counter App - view


import Html.Events exposing (..)

view : Model -> Html Msg
view model =
  div []
    [ p [] [ text (toString model) ]
    , button [ onClick Decrement ] [ text "-" ]
    , button [ onClick Increment ] [ text "+" ]
    ]

Counter App - update


update : Msg -> Model -> Model
update msg model =
  case msg of
    Increment -> model + 1
    Decrement -> model - 1

Elm Architecture Overview

This is the Elm Architecture

  • Model (just a type)
  • view: Model -> Html Msg
  • update: Msg -> Model -> Model

That's all there is.

(nearly)

But What About Side Effects?

  • AJAX
  • WebSockets
  • Randomness
  • ...

Problem: How do we communicate with the world outside Elm in when everything is purely functional? How do we manage side effects?

The Complete Elm Architecture.

  • Commands (Cmd): Elm apps execute side effects via Commands
  • Subscriptions (Sub): Elm apps receive things via Subscriptions (think web sockets)
  • Model
  • view: Model -> Html Msg
  • update: Msg -> Model -> (Model, Cmd Msg)
